A two-week data-engineering sprint to build a normalized PostgreSQL database
from the Kanto/Johto/Hoenn Pokemon dataset. The post walks through the full ETL
workflow, including schema design with primary and foreign keys for Pokemon,
types, abilities, generations, and moves.

The analytical excuse for all of this — which type has the strongest base stats —
becomes the test query the schema has to support.
