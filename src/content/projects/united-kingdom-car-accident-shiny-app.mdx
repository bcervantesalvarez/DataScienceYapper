## How it works

The dashboard is an **R Shiny** application backed by the UK Department for Transport's
historical accident records. Data is filtered reactively through `dplyr` pipelines and
rendered with `plotly` for the charts and `leaflet` for the map view.

Notable design decisions:

- **Defaults that surface signal first.** The landing view aggregates the full timeline
  so visitors see the headline trend before they touch a single control.
- **Decoupled filter and display state.** Filter changes update a reactive subset, and
  the visualizations subscribe to that subset rather than re-querying.
- **Cached aggregates.** Expensive grouped summaries are computed once on app start so
  subsequent filter changes feel instant.

## When this pattern fits

A dashboard like this is the right call when stakeholders need to ask their own
questions of a dataset, not just consume a pre-built chart deck. Pair it with an
exportable data download so analysts who want to take the underlying numbers further
can do so without asking for a re-run.
