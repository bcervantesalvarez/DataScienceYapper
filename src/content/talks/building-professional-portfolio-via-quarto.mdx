A practical talk on building a portfolio site with Quarto and GitHub Pages.
The argument: a portfolio is the single highest-leverage piece of personal
infrastructure for a data scientist, and the Quarto-on-Pages combination removes
most of the excuses for not having one.

The session covers project structure, deploying with GitHub Actions, and how to
organize projects, blog posts, and talks so the site actually scales as your body
of work grows.
